## 3. DHT11 温湿度模块

### 3.1 模块介绍

DHT11 是低速温湿度数字传感器，内部包含湿敏元件、NTC 温度测量元件和 8 位控制芯片。它不是 UART、I2C 或 SPI，而是单总线协议，只需要一个 MCU IO。

| 参数 | 说明 |
|---|---|
| 工作电压 | 3.3 V 到 5.5 V，野火模块接口按 3V3 标注 |
| 湿度范围 | 5-95%RH |
| 湿度精度 | 约 ±5%RH |
| 温度范围 | -20 到 60 摄氏度 |
| 温度精度 | 约 ±2 摄氏度 |
| 数据宽度 | 40 bit，湿度整数/小数、温度整数/小数、校验和 |
| 采样间隔 | 每次采集间隔需大于 2 秒 |

### 3.2 电路说明

DHT11 的 DATA 引脚通过上拉电阻保持空闲高电平。MCU 读取时先把 DATA 拉低超过 18 ms 发起开始信号，然后释放总线，DHT11 再输出响应脉冲和 40 bit 数据。

本项目把 `PB12` 配置成可切换模式：

| 阶段 | GPIO 模式 | 作用 |
|---|---|---|
| 起始信号 | 开漏输出 | MCU 拉低 DATA 约 20 ms |
| 释放总线 | 开漏输出写高后切输入 | 让上拉电阻把 DATA 拉高 |
| 接收数据 | 上拉输入 | DHT11 控制 DATA 输出 40 bit |

### 3.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 A SENSOR |
| 数据引脚 | `PB12` |
| 供电 | `3V3` |
| 代码入口 | `DHT11_Read()` |
| 采样控制 | `DHT11_PERIOD_MS = 2100` |
| 错误上报 | `STATUS_DHT_ERROR` 写入数据帧 `status` |

虽然板 A 每秒发送一帧，DHT11 并不是每秒读取。代码会保证两次 DHT11 读取至少间隔 2100 ms；由于帧周期是 1000 ms，所以实际温湿度大约每 3 秒刷新一次。

### 3.4 注意事项

- 不要把 DHT11 当成高速传感器连续读取，否则容易返回旧值或失败。
- 线太长时单总线容易受干扰，建议短线、共地、必要时增强上拉。
- DHT11 只适合演示级温湿度检测，不适合高精度测量。

## 4. MQ135 空气检测模块

### 4.1 模块介绍

MQ135 是加热型气体传感器模块，常用于空气质量趋势检测。它对氨气、硫化物、苯系蒸气等污染气体有响应，但本项目没有做 ppm 标定，只使用 ADC 原始值作为趋势和阈值判断依据。

| 参数 | 说明 |
|---|---|
| 推荐供电 | 5 V |
| 输出 | AO 模拟量，DO 比较器数字量 |
| 本项目使用 | AO 模拟量 |
| 预热 | 建议至少 5 分钟，稳定性更好的测试需要更长预热 |
| 标定 | 需要结合负载电阻、环境和曲线单独标定 |

### 4.2 电路说明

MQ135 模块通常包含 MQ135 敏感元件、加热电路、负载电阻和 LM393 比较器：

| 电路部分 | 作用 |
|---|---|
| 加热丝 | 让敏感材料进入工作温度 |
| 敏感电阻 | 随气体浓度变化而改变阻值 |
| AO 输出 | 反映传感器分压后的模拟电压 |
| LM393 比较器 | 将模拟量与电位器阈值比较，输出 DO |
| 电位器 | 调整 DO 触发阈值 |

本项目只读取 AO，因此 DO 阈值旋钮不会影响 MCU 采样值，但会影响模块板上的 DO 指示或 DO 输出。

### 4.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 A SENSOR |
| AO 接线 | `PA4 / ADC1_CH4` |
| VCC | `5V` |
| GND | `GND` |
| 代码入口 | `ADC1_ReadChannel(4)` |
| 数据处理 | 指数滑动平均，`mq135_avg = (old * 3 + raw) / 4` |
| 显示/报警 | 板 B 按阈值档位判断空气预警 |

### 4.4 注意事项

- `PA4` 是 ADC 输入，绝对不能输入超过 3.3 V 的模拟电压。
- MQ 模块资料中提到 AO 可能低于 4 V，但对 STM32 ADC 来说仍必须实测并保证最高不超过 3.3 V。
- 如果 AO 可能超 3.3 V，需要加分压或限幅电路。
- 当前代码显示的是 12 位原始 ADC 值，不是 ppm。

## 5. MQ2 烟雾检测模块

### 5.1 模块介绍

MQ2 是加热型烟雾和可燃气体传感器模块，对烟雾、丙烷、甲烷、乙醇、天然气等有响应。本项目用它做烟雾/可燃气体趋势检测和危险报警。

| 参数 | 说明 |
|---|---|
| 推荐供电 | 5 V |
| 输出 | AO 模拟量，DO 比较器数字量 |
| 本项目使用 | AO 模拟量 |
| 预热 | 建议至少 5 分钟 |
| 标定 | 当前项目未换算 ppm，需要按现场标定阈值 |

### 5.2 电路说明

MQ2 模块电路结构与 MQ135 类似，包含加热丝、敏感电阻、模拟分压输出、LM393 比较器和电位器。AO 输出反映模拟电压变化，DO 输出是比较器阈值结果。

本项目把 MQ2 和 MQ135 分开接两个 ADC 通道：

| 模块 | 项目 ADC 通道 | 原因 |
|---|---|---|
| MQ135 | `PA4 / ADC1_CH4` | 空气质量趋势 |
| MQ2 | `PA5 / ADC1_CH5` | 烟雾/可燃气体趋势 |

### 5.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 A SENSOR |
| AO 接线 | `PA5 / ADC1_CH5` |
| VCC | `5V` |
| GND | `GND` |
| 代码入口 | `ADC1_ReadChannel(5)` |
| 数据处理 | 指数滑动平均，`mq2_avg = (old * 3 + raw) / 4` |
| 显示/报警 | 板 B 按 warning/danger 阈值判断烟雾状态 |

### 5.4 注意事项

- `PA5` 是 ADC 输入，AO 必须限制在 0-3.3 V。
- MQ2 加热时电流较大，建议用稳定的 5 V 电源，不要让 5 V 供电造成板子复位。
- 烟雾测试要注意安全，演示可以用少量酒精挥发或安全烟源，避免明火靠近电路。

## 6. 火焰检测模块

### 6.1 模块介绍

火焰检测模块通常使用红外接收元件检测火焰或强红外光，野火资料中说明检测波长大致在 760-1100 nm。模块一般提供 AO 模拟量和 DO 数字量。

| 参数 | 说明 |
|---|---|
| 供电 | 3.3 V 或 5 V，按模块说明选择 |
| 输出 | AO 模拟量，DO 数字量 |
| 本项目使用 | DO 数字量 |
| 触发极性 | 检测到火焰强度超过阈值时 DO 输出低电平 |
| 阈值调节 | 通过模块电位器调节 DO 触发点 |

### 6.2 电路说明

模块电路一般由红外接收器、比较器和电位器组成。传感器信号与电位器设定阈值比较后输出 DO。野火资料说明 DO 的逻辑是：

| 状态 | DO 输出 |
|---|---|
| 未达到阈值 | 高电平 |
| 火焰或强红外达到阈值 | 低电平 |

### 6.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 A SENSOR |
| DO 接线 | `PB13` |
| VCC | `3V3/5V`，按模块要求 |
| GND | `GND` |
| 代码读取 | `HAL_GPIO_ReadPin(FLAME_PORT, FLAME_PIN)` |
| 触发判断 | `GPIO_PIN_RESET` 表示 `flame = 1` |
| 报警优先级 | 火焰触发属于 danger，板 B 红灯和快速蜂鸣 |

### 6.4 注意事项

- 如果模块 VCC 接 5 V，要确认 DO 输出对 STM32 `PB13` 数字输入安全。
- `PB13` 是 STM32F103 的 5V tolerant 数字 IO，但不要把 5 V 接入 ADC 类型引脚。
- 模块对太阳光、强红外遥控器等也可能响应，演示前要调电位器。

## 7. 0.96 寸 IIC OLED 模块

### 7.1 模块介绍

本项目使用 0.96 寸 IIC OLED，控制芯片为 SSD1306，分辨率 128x64。模块通过 I2C 接口接入，但本项目没有启用 STM32 硬件 I2C，而是用 `PB6/PB7` 做软件 I2C。

| 参数 | 说明 |
|---|---|
| 分辨率 | 128 x 64 |
| 控制器 | SSD1306 |
| 通信接口 | I2C |
| 供电 | 3.3 V |
| 地址 | 7-bit `0x3C`，发送字节地址 `0x78` |

### 7.2 电路说明

OLED 模块引脚为 VCC、GND、SCL、SDA。I2C 总线空闲为高电平，SCL/SDA 需要上拉。野火 OLED 模块默认带上拉，代码中也把 `PB6/PB7` 配成开漏输出并带上拉。

### 7.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 B MONITOR |
| SCL | `PB6` |
| SDA | `PB7` |
| VCC | `3V3` |
| GND | `GND` |
| 代码入口 | `OLED_Init_Custom()`、`OLED_Command()`、`OLED_Data()` |
| 显示刷新 | `Monitor_UpdateDisplay()` |

### 7.4 注意事项

- 扩展板 OLED 插座正好映射到 `PB6/PB7`，可以直插。
- 部分野火 OLED 独立例程可能使用 `PB10/PB11`，本项目不要照那个接，因为 `PB10/PB11` 已给 USART3。
- 当前软件 I2C 驱动没有做 ACK 错误检测，OLED 不接或地址错误时不会有明确错误码，只会无显示。

## 8. 有源蜂鸣器模块

### 8.1 模块介绍

有源蜂鸣器内部自带振荡驱动，只要给 SIG 一个有效电平就能响，不需要 MCU 输出 PWM。项目用它做本地报警声音提示。

| 参数 | 说明 |
|---|---|
| 类型 | 有源蜂鸣器 |
| 供电 | 野火模块按低压供电使用，项目接 3.3 V |
| 控制 | SIG 高电平响 |
| 本项目引脚 | `PB8` |

### 8.2 电路说明

有源蜂鸣器模块一般包含蜂鸣器本体和三极管驱动。MCU 输出高电平到 SIG 后，驱动管导通，蜂鸣器得电发声；MCU 输出低电平时关闭。

### 8.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 B MONITOR |
| SIG | `PB8` |
| VCC | `3V3` |
| GND | `GND` |
| 代码入口 | `Buzzer_Set()` |
| 报警节奏 | `Monitor_UpdateAlarm()` 按 normal/warn/danger/node lost 状态控制 |

### 8.4 注意事项

- 野火有源蜂鸣器模块不要接 5 V，`WIRING.md` 已按 3.3 V 固定说明。
- 如果换用其他蜂鸣器模块，先确认它是有源还是无源。无源蜂鸣器需要 PWM，本项目默认不适配。

## 9. W25Q64 Flash 模块

### 9.1 模块介绍

W25Q64 是 64 Mbit SPI NOR Flash，容量 8 MByte。本项目把它作为板 B 的可选历史记录存储器。不接 W25Q64 时，监测、显示、报警功能仍能正常运行。

| 参数 | 说明 |
|---|---|
| 容量 | 64 Mbit = 8 MByte |
| 供电 | 2.7-3.6 V，项目接 3.3 V |
| 接口 | 标准 SPI |
| 页大小 | 256 Byte |
| 扇区大小 | 4 KB |
| 常用指令 | JEDEC ID `0x9F`，读状态 `0x05`，写使能 `0x06`，页写 `0x02`，扇区擦除 `0x20` |
| 典型 JEDEC ID | Winbond 常见为 `EF 40 17` |

### 9.2 电路说明

模块引出了标准 SPI 四线和片选：

| 模块引脚 | 说明 |
|---|---|
| CS | 片选，低电平有效，模块默认上拉 |
| CLK/SCK | SPI 时钟 |
| DI/SI/MOSI | 主机输出，从机输入 |
| DO/SO/MISO | 主机输入，从机输出 |
| VCC | 3.3 V 电源 |
| GND | 地 |

WP 和 HOLD 通常在模块上拉到 VCC，不单独引出或不使用。项目使用标准单线 SPI，不使用 Dual/Quad SPI。

### 9.3 项目接法与代码

| 项目 | 内容 |
|---|---|
| 所在节点 | 板 B MONITOR，可选 |
| CS | `PB12` |
| SCK | `PB13` |
| MISO | `PB14` |
| MOSI | `PB15` |
| VCC | `3V3` |
| GND | `GND` |
| 代码入口 | `Flash_Init_Custom()`、`Flash_LogFrame()` |
| 检测方式 | 读取 `0x9F` JEDEC ID |
| 日志策略 | 初始化后擦除 sector 0，循环写入 20 字节记录 |

### 9.4 与扩展板的关系

扩展板上的 W25Q64/SPI1 直插接口连接的是：

```text
PA4 / PA5 / PA6 / PA7
```

本项目 W25Q64 使用的是：

```text
PB12 / PB13 / PB14 / PB15
```

因此本项目不能把 W25Q64 直接插在扩展板 SPI1 直插座上，必须用杜邦线接到 SPI2 引脚。这样设计是因为 `PA4/PA5` 已分别给 MQ135/MQ2 的 ADC 模拟输入使用。

### 9.5 注意事项

- W25Q64 严禁接 5 V。
- 项目初始化时会擦除 Flash sector 0，如果该模块里有重要数据，不要直接接入运行。
- 扩展板自带 PB15 按键与 `PB15` 同网，如果板 B 使用 W25Q64 MOSI，不要按这个按键。
- 当前 SPI 底层收发是演示型轮询实现，初始化和日志写入已能满足本项目，但工业级应用应增加更完整的超时和错误恢复。

## 10. 双板 USART3 通信线

### 10.1 连接说明

两块板之间不是通过 Wi-Fi 或 USB 通信，而是用 MCU 的 USART3 TTL 电平直连。连接时必须 TX/RX 交叉，并且共地。

| 板 A SENSOR | 板 B MONITOR | 说明 |
|---|---|---|
| `PB10 / USART3_TX` | `PB11 / USART3_RX` | 板 A 发送采集帧到板 B |
| `PB11 / USART3_RX` | `PB10 / USART3_TX` | 预留未来 ACK 或心跳 |
| `GND` | `GND` | 必须共地 |

串口参数固定为：

```text
115200 8N1
```

### 10.2 数据帧说明

板 A 每秒发送 13 字节数据帧：

```text
AA 55 LEN TEMP HUMI MQ135_H MQ135_L MQ2_H MQ2_L FLAME SEQ STATUS CHECKSUM
```

| 字段 | 说明 |
|---|---|
| `AA 55` | 帧头，用于重同步 |
| `LEN` | 负载长度，固定 `9` |
| `TEMP/HUMI` | DHT11 温湿度整数值 |
| `MQ135_H/L` | MQ135 12 位 ADC 值拆成高/低字节 |
| `MQ2_H/L` | MQ2 12 位 ADC 值拆成高/低字节 |
| `FLAME` | `1` 表示火焰触发 |
| `SEQ` | 帧序号 |
| `STATUS` | `bit0 = DHT11 读取错误` |
| `CHECKSUM` | `LEN + payload` 的低 8 位 |

### 10.3 调试现象

| 现象 | 可能原因 |
|---|---|
| 板 A 有 `[SENSOR]` 日志，板 B 一直 `NODE LOST` | USART3 没交叉、GND 未共地、板 B 烧错固件 |
| 板 B 偶尔丢帧但能恢复 | 串口噪声或接触不良，帧头重同步会自动恢复 |
| 板 B OLED 数值不变 | 板 A 未发送、串口线断开、帧校验失败 |
| 板 A 无日志 | USART1 调试串口未连接或波特率不对 |

## 11. 官方默认接线与本项目接线差异

| 模块 | 官方例程中常见接线 | 本项目接线 | 必须按项目接线的原因 |
|---|---|---|---|
| DHT11 | `PB0` 或 `PB12` | `PB12` | 与当前固件 `DHT11_PORT/PIN` 一致 |
| OLED | `PB10/PB11` 或扩展板 `PB6/PB7` | `PB6/PB7` | `PB10/PB11` 已用于 USART3 |
| 蜂鸣器 | `PA6`、`PB9` 等 | `PB8` | 板 B 报警输出固定在 `PB8` |
| 火焰 DO | `PA11`、`PA5` 等 | `PB13` | `PA11/PA12` 保留 USB，`PA5` 给 MQ2 ADC |
| MQ135 AO | `PA4` | `PA4` | 与官方一致 |
| MQ2 AO | 可能也默认 `PA4` | `PA5` | 两个 MQ 模块需要两个 ADC 通道 |
| W25Q64 | SPI1 `PA4/PA5/PA6/PA7` | SPI2 `PB12/PB13/PB14/PB15` | `PA4/PA5` 已给 MQ ADC |

## 12. 单模块测试建议

| 测试对象 | 推荐步骤 | 通过标准 |
|---|---|---|
| DHT11 | 板 A 上电后观察 `[SENSOR]` 日志 | 温湿度约每 3 秒更新，`status` 不长期为 `0x01` |
| MQ135 | 预热后靠近酒精/空气变化源 | `mq135` ADC 值有明显变化 |
| MQ2 | 预热后靠近安全烟雾/酒精挥发源 | `mq2` ADC 值有明显变化 |
| 火焰模块 | 调电位器，用安全光源靠近 | `flame=1`，板 B 进入 danger |
| OLED | 烧录板 B 固件后上电 | 显示状态页和数值 |
| 蜂鸣器 | 触发 danger 或 node lost | 蜂鸣器按报警节奏响 |
| W25Q64 | 接好 SPI2 后上电 | 日志打印 W25Q ID，不接时系统仍运行 |
| USART3 | 两板交叉连接并共地 | 板 B 打印 `[MONITOR] rx`，OLED 数值刷新 |
