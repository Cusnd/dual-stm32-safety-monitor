## Core Directory / Core 目录

| Path / 路径 | Purpose / 用途 |
|---|---|
| `Core/Src/main.c` | Main application logic for both nodes. / 两个节点共用的主要应用逻辑。 |
| `Core/Src/gpio.c` | CubeMX-style GPIO initialization for on-board keys and RGB LED. / CubeMX 风格的 GPIO 初始化，配置板载按键和 RGB LED。 |
| `Core/Src/stm32f1xx_it.c` | Interrupt handler file generated by STM32CubeMX. USART3 handler is implemented in `main.c`. / CubeMX 生成的中断文件；USART3 处理函数在 `main.c` 中实现。 |
| `Core/Src/stm32f1xx_hal_msp.c` | HAL low-level MSP initialization. / HAL 底层 MSP 初始化文件。 |
| `Core/Src/syscalls.c` | Minimal system call stubs for `printf` and embedded C library support. / 支持 `printf` 和嵌入式 C 库的最小系统调用桩。 |
| `Core/Src/sysmem.c` | Minimal heap implementation used by the C runtime. / C 运行库使用的最小堆实现。 |
| `Core/Src/system_stm32f1xx.c` | ST system clock support code. / ST 提供的系统时钟支持代码。 |
| `Core/Inc/main.h` | Shared board pin macros and common declarations. / 公共板载引脚宏和声明。 |
| `Core/Inc/gpio.h` | Header for `gpio.c`. / `gpio.c` 的头文件。 |
| `Core/Inc/stm32f1xx_hal_conf.h` | HAL module configuration. / HAL 模块配置文件。 |
| `Core/Inc/stm32f1xx_it.h` | Interrupt header generated by CubeMX. / CubeMX 生成的中断头文件。 |

For most feature work, beginners should mainly edit `Core/Src/main.c`, `Core/Inc/main.h`, and sometimes `Core/Src/gpio.c`.

初学者做功能扩展时，大多数情况下主要修改 `Core/Src/main.c`、`Core/Inc/main.h`，偶尔修改 `Core/Src/gpio.c`。

## Drivers Directory / Drivers 目录

`Drivers/` contains vendor code from ST:

`Drivers/` 里主要是 ST 官方代码：

| Path / 路径 | Purpose / 用途 |
|---|---|
| `Drivers/CMSIS/` | Cortex-M and STM32F1 device definitions. / Cortex-M 内核和 STM32F1 芯片定义。 |
| `Drivers/STM32F1xx_HAL_Driver/` | STM32F1 HAL source and headers. / STM32F1 HAL 驱动源码和头文件。 |

Usually you should not edit files under `Drivers/`. Treat them as library code.

一般不建议修改 `Drivers/` 下的文件，把它们当作库代码使用即可。

## Build Files / 构建文件

| File / 文件 | Purpose / 用途 |
|---|---|
| `CMakeLists.txt` | Defines the firmware target, role selection, and HEX/BIN generation. / 定义固件目标、节点角色选择和 HEX/BIN 生成。 |
| `CMakePresets.json` | Provides `SensorDebug` and `MonitorDebug` presets. / 提供 `SensorDebug` 与 `MonitorDebug` 两个预设。 |
| `cmake/gcc-arm-none-eabi.cmake` | ARM GCC toolchain configuration. / ARM GCC 工具链配置。 |
| `cmake/stm32cubemx/CMakeLists.txt` | Lists CubeMX-generated sources and HAL/CMSIS include paths. / 列出 CubeMX 生成源码以及 HAL/CMSIS 头文件路径。 |
| `STM32F103XX_FLASH.ld` | Linker script that describes flash and RAM layout. / 描述 Flash 和 RAM 布局的链接脚本。 |
| `startup_stm32f103xb.s` | Startup assembly file with vector table and reset entry. / 启动汇编文件，包含中断向量表和复位入口。 |

Build commands:

构建命令：

```powershell
cmake --preset SensorDebug
cmake --build --preset SensorDebug

cmake --preset MonitorDebug
cmake --build --preset MonitorDebug
```

Outputs:

输出产物：

```text
build/SensorDebug/Fire_F103_sensor.hex
build/MonitorDebug/Fire_F103_monitor.hex
```

## Ignored Local Files / 被忽略的本地文件

The following paths are intentionally ignored by Git:

以下路径会被 Git 忽略：

| Path / 路径 | Why ignored / 忽略原因 |
|---|---|
| `assets/` | Local PDFs, screenshots, reference materials, and images. / 本地 PDF、截图、参考资料和图片。 |
| `build/` | CMake build outputs, object files, ELF/HEX/BIN products. / CMake 构建输出、目标文件和 ELF/HEX/BIN 产物。 |
| `.idea/` | CLion/JetBrains local IDE settings. / CLion/JetBrains 本地 IDE 设置。 |
| `MDK-ARM/Fire_F103/` | Keil generated intermediate/output files. / Keil 生成的中间文件和输出文件。 |

This keeps the public repository focused on source code and useful documentation.

这样可以让公开仓库只保留源码和有价值的说明文档。

## Where To Change Things / 常见修改入口

| Goal / 目标 | Edit here / 修改位置 |
|---|---|
| Add a new sensor / 添加新传感器 | `Core/Src/main.c`, sensor pin macros, `Sensor_GPIO_Init()`, `Sensor_App_Run()` |
| Add a new displayed value / 添加显示数值 | `SensorFrame`, `Frame_Encode()`, `Frame_Decode()`, `Monitor_UpdateDisplay()` |
| Change alarm thresholds / 修改报警阈值 | `k_threshold_profiles` in `Core/Src/main.c` |
| Change OLED pages / 修改 OLED 页面 | `Monitor_UpdateDisplay()` |
| Change buzzer or LED behavior / 修改蜂鸣器或灯效 | `Monitor_UpdateAlarm()`, `LED_Set()`, `Buzzer_Set()` |
| Change board-to-board frame format / 修改双板通信帧格式 | `FRAME_PAYLOAD_LEN`, `SensorFrame`, `Frame_Encode()`, `Frame_Decode()`, docs |
| Change build output names / 修改构建产物名称 | `CMakeLists.txt` |

When changing the frame format, update both sender and receiver at the same time. Otherwise Board B will reject Board A's frames.

修改通信帧格式时，必须同时改发送端和接收端；否则板 B 会因为长度或校验不匹配而丢弃板 A 的数据帧。
